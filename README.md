プログラミングって何？って人が、簡単なアプリを作れるようになるまでの道標（新入生・文系学生向け）

## 1. まえがき

堂々と「AIコーディング最新動向まとめ（2026年◯月）」って記事を書こうとしたけど、流行に追いつけずどんどん取り残されている老害になりつつあるので、断念。代わりにプログラミング経験がない人や、ちょっと授業でかじったくらいの初心者に向けて、情電数系の学生が日夜にらめっこしてる謎の黒い画面の正体を見ていきましょう。

※財政難に伴い、3月末でChatGPTを解約してしまった筆者が、3月31日23時現から急ピッチで執筆を開始しています。

## 2. CordingAgentとは

CordingAgentとは、AIがPC内のファイルを直接見ながら、自律的にプログラミングを進めてくれるというものです。ChatGPTやGeminiのチャット画面に「”いい感じに”コード書いて」って投げて、吐き出されたコードをコピペ。実行したらエラーまみれで動かない、チャット画面に戻って

<aside>
🤬「動かんやないか、ふざけるな！書き直せ！！」
</aside>

と、AIに罵詈雑言をまくしたててバグ修正させる、みたいな運用をしている人をよく見かけますが、CordingAgentではその無駄なラリーが不要です。なんなら、コーディングに最適化されたモデルが常駐しているので、チャットのAIよりも幾分優秀です。（課金の有無など、状況による）

これら、CordingAgentの多くはエディタの拡張機能として実装されており、まずはエディタのインストールが必要です。その他にもブラウザ版やCLI版もありますが、本稿では割愛します。

## 3. エディタとは

VSCode（Visual Studio Code）を聞いたことはありますか？これは、最もフィジカルで最もプリミティブで最もフェティッシュなエディタの1つです。情報系の学生が毎日、気が狂ったように見つめている黒い画面のことです。分かりやすく言うと、**Windows標準アプリ「📝メモ帳」に100本くらいヒゲが生えた、超高機能なメモ帳**です。プログラミングや開発をするときは、このようなエディタを用いることが多いです。他にも、有名なもので言えば以下のようなものがあります。

- VSCode（Microsoft）：https://code.visualstudio.com/
- Antigravity（Google）：https://antigravity.google/
- Cursor（Anysphere）：https://cursor.com/
- den（K16858）：https://github.com/K16858/den-editor
- Vim（Bram Moolenaar）：https://www.vim.org/
- ~~Emacs（GNU）：https://www.gnu.org/software/emacs/~~

上3つに関しては特に高機能で、今回紹介するCordingAgentが使えます。本稿では、VSCodeとAntigravityで進めて行きます。

まずは、要件定義を作成します。

```jsx
# Role
あなたはGUIアプリケーション開発に精通したPythonエンジニアです。
ユーザーが直感的にPDFを編集できるデスクトップアプリを開発してください。

# Goal
以下の4つの機能を備えた、マウス操作主体のPDF編集アプリ（Python）を作成する。
1. PDFの結合 (Merge)
2. PDFの分割 (Split)
3. 特定ページの削除 (Delete Pages)
4. 特定ページの抽出 (Extract Pages)

# Technical Stack
- Language: Python 3.10+
- GUI Framework: PyQt6 (または PySide6)
- PDF Library: PyMuPDF (fitz) または pikepdf
- Layout: モダンでクリーンなUIデザイン

# Requirements & UI/UX
1. **直感的な操作**:
   - PDFファイルをドラッグ＆ドロップで読み込めること。
   - 読み込んだPDFの各ページを「サムネイル画像」として一覧表示すること。
   - マウスでページを選択（クリックや範囲選択）して、削除や抽出の対象を選べること。
   - 結合の際は、ファイルやページの順番をドラッグ＆ドロップで入れ替えられること。

2. **機能詳細**:
   - 結合：複数のPDFを読み込み、一つのファイルにまとめる。
   - 分割：1つのPDFをページごとに、あるいは指定範囲で別々のファイルに分ける。
   - 削除：サムネイル一覧から不要なページを選択して「削除」ボタンで取り除く。
   - 抽出：選択したページだけを新しいPDFとして保存する。

3. **出力**:
   - 処理後のPDFを保存する際のファイルダイアログを表示すること。

# Deliverables
1. `requirements.txt` (必要なライブラリ)
2. `main.py` (実行可能なメインコード)
3. 必要であれば、ロジックとUIを分けたモジュール構成
```

一方で、問題点として**ドチャクソ料金が高い**です。GitHubの新規受付停止でワチャワチャしてるのにも少し関係しています。正直今後実用するのは厳しいでしょう。次点で、Codexが優秀なので、ここらで手を打つのがいいと思います。

## 7. 最後に

よく、「初心者でも◯日でアプリをリリースできる！」とか言ってる怪しい人の投稿や勉強会などが横行していますが、絶対にやめてください。あなたは何でもできるようになったと、錯覚しているだけです。（自戒）ちゃんとプロトコルスタックやセキュリティに精通した上で、AIのコードを必ず自分で理解してチェックし、第三者のレビューなどを必ず受けてください。

具体名の提示は避けますが、ある日某学生がAIを用いてアプリ開発したという情報が入ってきました。実際に使ってみるとUIがバグまみれの上に、バックエンドのコードを確認してみると、API Keyは丸見え、謎のレスポンスメッセージが大量に出ている上に、挙動不審なネットワーク使用などなど、穴だらけでした。直接開発メンバーとミーティングを設定して、初対面でしたが「こんな状態で収益化しようとするな」と伝えるに至りました。
